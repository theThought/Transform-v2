# Scope
Researcher documentation is focused on the different response types that they can use within the Transform platform.  Within the platform the question is considered the way the request is presented to the user where as the response is considered the way that the respondent provides an answer.  To this end, what have previously been called question types are now called response types (ways in which the respondent can respond to a question).

# Template
Each Response type is presented in a similar way to aid in rapidly finding the reuired information.  Within these pages the high-level definition of the response type is provided.  Embedded within this are links to more detailed defiitions that show how the response type is constructed.
To produce a consisstent design across response types each type has been broken down into a hierarchical structure of components.  These components are defined and then shared across response types.

## Structure and Variations
After a brief description of the response type the first main section defines the repsonse types stucture. This details the components that a response type is made of.  Components are hierarchical and so some components may be made up of other components.  This is not shown in the structure, but diving into the details of each component provides access to the lower levels of component definition.  This information is not genearally required to implement a response type.
Accompanying the structure is detail of any variations that can be found within the response type, different ways in which the response type can be laid out / composed.
