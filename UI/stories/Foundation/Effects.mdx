# Effects

Effect tokens define shadows and motion behaviour. They communicate depth, elevation, and interactivity in a consistent way across all components.

## Shadow

Shadow tokens apply depth and visual feedback to components. They are used to communicate interactive state changes such as focus or activation.

## Transition

Transition tokens define the timing and easing of state changes across all components. A global rule applies `--transition-base` to every element, so interactive state changes — hover, focus, active, disabled — animate consistently without per-component declarations.

The composite token covers only properties that change on interaction. Layout properties such as `width`, `height`, `padding`, and `transform` are intentionally excluded to prevent unintended animation.
