# Color Semantics

Semantic tokens map raw color primitives to specific UI roles and purposes. Rather than referencing color values directly, components and layouts consume these tokens — ensuring consistent application, simplified theming, and a clear single source of truth for every color decision across the design system.

## Background

Tokens for surface and container fill colors. Apply these to page backgrounds, cards, containers, and other structural areas to establish visual hierarchy and communicate element states.

### Neutral

<ColorPalette>
  <ColorItem title="--color-bg-neutral-primary"   colors={{ 'var(--base-color-neutral-10)': '#FFFFFF' }} />
  <ColorItem title="--color-bg-neutral-secondary" colors={{ 'var(--base-color-neutral-20)': '#FAFAFA' }} />

</ColorPalette>
